Reverse DNS

Tags: DNS PTR Record Spam

안녕하세요, 오늘은 Reverse DNS에 대해 알아보는 시간을 가지도록 하겠습니다.

Reverse DNS

Reverse DNS는 Reverse DNS Lookup, rDNS 라고도 부릅니다. 일반적으로 Domain Name을 IP address에 Mapping하는 반면, 이 Reverse DNS는 그와 반대로 IP Address를 Domain Name에 Mapping합니다. 이 때 연관되는 기술이 있는데요, Inverse Domain과 DNS Zone File의 Resource Records 유형 중 하나인 PTR Record를 이용하여 IP Address를 통한 역질의를 합니다.

Inverse Domain

Inverse Domain은 Reverse Domain이라고도 부르며 Name Server에서 설정하는 특수한 Domain입니다. 최상위 Domain Name은 arpa로 설정되어 있으며 그 하위에는 in-addr, e164, ip6, uri, urn 등이 존재합니다. Inverse Domain의 Zone File은 IP Address의 Octet을 기준으로 노드를 형성하고 트리 형태로 이루어진 자료구조로 관리됩니다. 작동 원리는 다음과 같이 IPv4를 예시로 들어보겠습니다.

작동 원리

  1. 역질의 시 네임 서버에서 역질의하게 되는 상황에 맞게 하위 도메인을 고릅니다. (여기서는 in-addr)
  2. IP Address의 Octet을 기준으로 거꾸로 배열하여 하위 도메인과 결합힙나다. (e.g. 4.3.2.1.in-addr.arpa)
  3. 해당 Zone File을 찾고 PTR Record 값을 읽어 호스트명을 응답합니다.

마무리

Reverse Domain은 이메일 서비스에서 스팸필터 기능으로 많이 사용되고 있습니다. 현재 시점으로 국내에서는 권장하고 있고 국외에서는 거의 필수적으로 도입하고 있습니다. 국외와 관련된 서비스를 고려하신다면 반드시 설정해야 할 사항인 것 같습니다.
